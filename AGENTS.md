## 理念与原则

| 原则     | 说明              | 类型   | 应用场景            |
| ------ | --------------- | ---- | --------------- |
| 单一来源   | 信息只在一处维护        | 价值观  | 索引由脚本生成，不手动编辑   |
| 自动化优先  | 能自动的绝不手动        | 价值观  | Git Hook 自动更新索引 |
| 目录即数据  | 目录结构决定内容        | 价值观  | 研究方向/研究主题/报告.md |
| DRY    | 不重复             | 检验标准 | 检查是否有重复章节/代码    |
| YAGNI  | 不创建无用内容         | 检验标准 | 删除冗余脚本和文档       |
| 幂等性    | 自动化脚本可重复执行，结果一致 | 检验标准 | 增量更新替代一次性操作混入脚本 |
| Git 原生 | 所有变更可追溯         | 价值观  | 版本管理是核心         |

## 协作经验

1. **质疑而非顺从** - 用原则审视建议必要性
2. **讨论而非独裁** - 质疑后说明理由，等待回应
3. **共识而非讨好** - 目标是达成正确结论

## 规则

- **冲突处理**：目标与规则冲突时，优先询问用户
- **确认优先**：遇到问题先确认现状（`git status`）
- **10 分钟规则**：同一问题尝试 10 分钟未解决，必须询问
- **善用工具，流程优先**：复杂任务拆解并行化，重复劳动流程化
- **提交确认**：每次提交前必须向用户确认，获得明确同意后才能提交
