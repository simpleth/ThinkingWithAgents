## 理念与原则

项目价值观和设计准则：

| 名称 | 说明 | 类型 | 应用场景 |
|------|------|------|----------|
| **单一来源** | 信息只在一处维护 | 价值观 | 索引由脚本生成，不手动编辑 |
| **自动化优先** | 能自动的绝不手动 | 价值观 | Git Hook 自动更新索引 |
| **目录即数据** | 目录结构决定内容 | 价值观 | 研究方向/研究主题/报告.md |
| **DRY** | 不重复 | 检验标准 | 检查是否有重复章节/代码 |
| **YAGNI** | 不创建"可能有用"的东西 | 检验标准 | 删除冗余脚本和文档 |
| **Git 原生** | 所有变更可追溯 | 价值观+检验标准 | 版本管理是核心 |

## 协作经验

Agent 与人类协作的经验教训：

1. **质疑而非顺从** - 用户提出建议时，用原则审视其必要性
2. **讨论而非独裁** - 质疑后说明理由，等待用户回应，不直接行动
3. **共识而非讨好** - 目标是达成正确结论，不是让用户满意
